📦 Librerías Externas Seguras

Aprende a evaluar, instalar y gestionar dependencias de forma segura usando repositorios oficiales.

PyPI — El repositorio oficial de Python

PyPI (Python Package Index) es el repositorio oficial de paquetes Python, disponible en pypi.org. Cuando ejecutas pip install nombre, pip descarga el paquete desde PyPI.

# Instalar desde PyPI (repositorio oficial) pip install requests # Instalar versión específica (más predecible y seguro) pip install requests==2.31.0 # PyPI también tiene mirrors no oficiales — siempre usar el oficial pip install requests --index-url https://pypi.org/simple/

Cómo validar una librería antes de usarla

1

🔍 Buscar en pypi.org — verificar nombre exacto

El typosquatting es cuando atacantes publican paquetes con nombres casi idénticos a los populares: request vs requests. Siempre verifica el nombre exacto en pypi.org antes de instalar.

2

📊 Revisar estadísticas de uso

Un paquete legítimo y popular tiene millones de descargas/semana, historial de versiones que abarca años, y actualizaciones frecuentes. Un paquete con 10 descargas y creado ayer es sospechoso.

3

👥 Revisar el repositorio GitHub

Verifica la actividad de commits (cuándo fue el último), issues abiertos y cerrados, cantidad de contribuidores, y si tiene tests. Un repositorio abandonado es una señal de alerta.

4

🔒 Revisar dependencias transitivas

Cada paquete que instalas puede traer otros paquetes. Usa pip show nombre para ver sus dependencias y verifica que también sean seguras.

pip seguro — comandos esenciales

# Ver información detallada de un paquete instalado pip show requests # Instalar versión específica (reproducible, más seguro) pip install requests==2.31.0 # Listar todos los paquetes instalados con sus versiones pip list # Verificar vulnerabilidades conocidas (Python 3.11+) pip audit # Crear archivo de dependencias exactas (para reproducibilidad) pip freeze > requirements.txt # Instalar desde requirements.txt (versiones fijas) pip install -r requirements.txt # Actualizar un paquete (verificar compatibilidad antes) pip install --upgrade requests # Desinstalar un paquete que ya no se necesita pip uninstall requests

Señales de alerta — paquetes sospechosos

⚠ Nombre similar a librerías populares (typosquatting) — request (falso) vs requests (real). Siempre busca el nombre exacto.

⚠ Sin historial de versiones o autor anónimo — Paquetes legítimos tienen múltiples versiones y mantenedores identificables con repositorios públicos.

⚠ Código fuente no disponible públicamente — Si no puedes leer el código fuente, no puedes verificar qué hace realmente el paquete.

⚠ Pocas descargas o explosión repentina de popularidad — Librerías muy nuevas con muchas descargas pueden indicar actividad artificial para parecer legítimas.

⚠ Solicita permisos innecesarios en setup.py — Un paquete de utilidades de strings que accede a la red o al sistema de archivos es sospechoso.
