🔍 Búsqueda y Ordenamiento

Implementa y comprende los algoritmos de búsqueda y ordenamiento en Python. Visualiza su comportamiento y complejidad.

busqueda_lineal.py 
def busqueda_lineal(lista, objetivo):
    """O(n) — recorre cada elemento."""
    for i, elemento in enumerate(lista):
        if elemento == objetivo:
            return i   # retorna el índice
    return -1        # no encontrado

numeros = [34, 7, 23, 32, 5, 62]
pos = busqueda_lineal(numeros, 23)
print(f"Encontrado en índice: {pos}")  # → 2

# Caso no encontrado
pos2 = busqueda_lineal(numeros, 99)
print(pos2)  # → -1

# ¿Cuándo usar?
# ✅ Lista no ordenada
# ✅ Lista pequeña (< 100 elementos)
# ❌ Lista grande y ordenada → usar binaria
def busqueda_binaria(lista, objetivo):
    """O(log n) — requiere lista ORDENADA."""
    izq, der = 0, len(lista) - 1
    while izq <= der:
        medio = (izq + der) // 2
        if lista[medio] == objetivo:
            return medio
        elif lista[medio] < objetivo:
            izq = medio + 1
        else:
            der = medio - 1
    return -1

ordenados = [5, 7, 23, 32, 34, 62]
pos = busqueda_binaria(ordenados, 32)
print(f"Encontrado en índice: {pos}")  # → 3

numeros = [34, 7, 23, 32, 5, 62]

# sort() — modifica la lista EN LUGAR (in-place)
numeros.sort()              # [5, 7, 23, 32, 34, 62]
numeros.sort(reverse=True)  # [62, 34, 32, 23, 7, 5]

# Ordenar por múltiples criterios
personas = [("Ana", 21), ("Luis", 19), ("Sofía", 21)]
personas.sort(key=lambda p: (p[1], p[0]))
# → [('Luis', 19), ('Ana', 21), ('Sofía', 21)]

# Diferencia clave:
# sort()   → modifica original (in-place)
# sorted() → devuelve nueva lista (funcional)
copia_ord = sorted(numeros)  # preserva numeros

Búsqueda Lineal

O(n) · sin requisitos · simple

Recorre cada elemento uno por uno hasta encontrar el objetivo. No requiere que la lista esté ordenada.

O(n)
Complejidad lineal
En el peor caso, recorre todos los n elementos. Con 1000 elementos: hasta 1000 comparaciones.

Búsqueda Binaria

O(log n) · lista ordenada · divide y vencerás

Divide el espacio de búsqueda a la mitad en cada paso. Mucho más eficiente que la lineal, pero requiere lista ordenada.

O(log n)
Complejidad logarítmica
Con 1000 elementos: máximo ~10 comparaciones. Con 1.000.000: ~20 comparaciones.
