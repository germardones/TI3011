🛡️ Colecciones Seguras

Uso seguro de listas, tuplas y diccionarios. Conoce los riesgos de mutabilidad y cómo prevenirlos con buenas prácticas defensivas.

listas_seguras.py 
# ── LISTAS SEGURAS ──────────────────────────────

# ✅ Verificar índice antes de acceder
notas = [6.5, 5.8, 7.0, 4.2]
idx = int(input("Índice: "))
if 0 <= idx < len(notas):
    print(notas[idx])
else:
    print("Índice fuera de rango.")

# ✅ Copiar lista, no crear referencia
original = [1, 2, 3]
copia    = original.copy()   # o list(original)
copia.append(4)
print(original)  # [1, 2, 3] — no cambia

# ✅ Nunca modificar la lista mientras se itera
items = [1, 2, 3, 4, 5]
# ❌ INSEGURO: for i in items: if i%2==0: items.remove(i)
pares = [i for i in items if i % 2 == 0]

# ✅ Validar antes de agregar
def agregar_nota(lista, nota):
    if not isinstance(nota, (int, float)):
        raise TypeError("La nota debe ser numérica.")
    if not (1.0 <= nota <= 7.0):
        raise ValueError("Nota entre 1.0 y 7.0")
    lista.append(nota)
# ── TUPLAS SEGURAS ──────────────────────────────

# ✅ Usar tuplas para datos inmutables (configuración)
DB_CONFIG = ("localhost", 5432, "mi_db")
host, port, nombre = DB_CONFIG

# ✅ Retornar múltiples valores con tupla
def procesar_nota(nota: float) -> tuple:
    if nota >= 4.0:
        return (True, "Aprobado", nota)
    return (False, "Reprobado", nota)

aprobado, estado, n = procesar_nota(5.5)

# ✅ Tupla como clave de diccionario (inmutable)
registro = {}
registro[("Ana", "TI3011")] = 6.5

# ✅ Proteger datos sensibles con namedtuple
from collections import namedtuple
Credencial = namedtuple("Credencial",
                         ["usuario", "rol"])
cred = Credencial("ana@inacap.cl", "alumno")
# cred.usuario → acceso legible
# cred.usuario = "x" → AttributeError (inmutable)
# ── DICCIONARIOS SEGUROS ────────────────────────

# ✅ Usar .get() con valor por defecto
usuario = {"nombre": "Ana", "rol": "alumno"}

# ❌ INSEGURO: puede lanzar KeyError
# email = usuario["email"]

# ✅ SEGURO: valor por defecto controlado
email = usuario.get("email", "no-registrado")

# ✅ Validar clave antes de acceder
if "rol" in usuario:
    print(f"Rol: {usuario['rol']}")

# ✅ Nunca guardar contraseñas en texto plano
# ❌ usuario["password"] = "admin123"
import hashlib
def hash_pass(pw: str) -> str:
    return hashlib.sha256(pw.encode()).hexdigest()

usuario["pw_hash"] = hash_pass("miPassword")

# ✅ Copiar dict antes de modificar
config_base = {"debug": False, "max_req": 100}
config_local = config_base.copy()
config_local["debug"] = True  # no afecta config_base

list — Mutable

[ ] · ordenada · mutable · permite duplicados

La mutabilidad permite modificar datos en runtime, lo que puede ser fuente de bugs y vulnerabilidades si no se controla.

tuple — Inmutable

( ) · ordenada · inmutable · más segura

La inmutabilidad garantiza que los datos no cambien accidentalmente. Ideal para configuración, credenciales de acceso y retorno de funciones.

dict — Clave-Valor

{ k: v } · mutable · acceso O(1) · claves únicas

Potentes pero con riesgos. Nunca almacenar contraseñas en texto plano; usar .get() para evitar KeyError; copiar antes de modificar.
