🚨 Errores y Excepciones

Captura y maneja errores de forma segura, evitando exponer información sensible del sistema. Aprende las mejores prácticas orientadas a seguridad.

excepciones_seguras.py 
# El usuario ingresa dos números para dividir
# ¿Qué pasa si escribe "hola" o divide entre 0?

# ── SIN manejo de errores ──────────────────────────
a = int(input("Numerador: "))    # crash si escribe "hola"
b = int(input("Denominador: "))  # crash si escribe "hola"
print(a / b)                      # crash si b == 0

# ── CON try/except ────────────────────────────────
try:
    a = int(input("Numerador: "))
    b = int(input("Denominador: "))
    resultado = a / b
    print(f"Resultado: {resultado}")
except ValueError:
    # el usuario escribió texto en vez de un número
    print("Error: ingresa solo números enteros.")
except ZeroDivisionError:
    # el usuario ingresó 0 como denominador
    print("Error: no se puede dividir entre cero.")
finally:
    # esto se ejecuta SIEMPRE, haya error o no
    print("--- Operación finalizada ---")
# raise lanza un error de forma intencional
# Se usa para validar datos ANTES de operar

def dividir(a, b):
    if not isinstance(a, (int, float)) or \
       not isinstance(b, (int, float)):
        raise TypeError("Ambos valores deben ser números.")
    if b == 0:
        raise ValueError("El denominador no puede ser cero.")
    return a / b

# Uso correcto
print(dividir(10, 2))   # → 5.0

# Estos llaman a raise internamente:
dividir("hola", 2)     # → TypeError
dividir(10, 0)         # → ValueError

# ── Excepción propia ──────────────────────────────
class DivisorNegativoError(Exception):
    pass

def dividir_positivo(a, b):
    if b < 0:
        raise DivisorNegativoError(
            "El divisor debe ser positivo.")
    return a / b
import logging

# Configurar logging para guardar en archivo
logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s [%(levelname)s] %(message)s',
    filename='calculadora.log'
)

def dividir(a, b):
    logging.debug(f"Intentando dividir {a} / {b}")
    try:
        resultado = a / b
        logging.info(f"División exitosa: {resultado}")
        return resultado
    except ZeroDivisionError:
        # registro técnico → al archivo (no al usuario)
        logging.error("División por cero: b=0")
        # mensaje genérico → al usuario
        print("No puedes dividir entre cero.")
        return None

dividir(10, 2)   # INFO  → log
dividir(10, 0)   # ERROR → log  +  print al usuario

# Niveles: DEBUG → INFO → WARNING → ERROR → CRITICAL
logging.warning("Resultado muy grande, revisar")
logging.critical("Fallo inesperado del sistema")

try / except

captura · manejo · recuperación

¿Qué es?

Bloque que intenta ejecutar código que puede fallar y captura el error si ocurre, evitando que el programa se detenga bruscamente.

¿Cuándo usarlo?

⌨️
El usuario ingresa datos que pueden ser de tipo incorrecto (ValueError)
➗
Se realiza una operación aritmética que puede ser inválida (ZeroDivisionError)
🌐
Se conecta a internet, lee archivos o accede a recursos externos que pueden fallar

Bloques disponibles

try
Código que puede lanzar una excepción
except
Código que se ejecuta solo si ocurre el error indicado
else
Código que se ejecuta solo si NO hubo ningún error
finally
Código que se ejecuta siempre, haya o no haya error
❌ Anti-patrón

Usar except Exception as e: print(e) captura cualquier error sin distinción y muestra detalles técnicos al usuario.

✅ Buena práctica

Captura ValueError y ZeroDivisionError por separado con mensajes claros para cada caso.

raise

lanzar · validar · fail fast

¿Qué es?

Lanza una excepción de forma intencional. Lo usa el programador para detener la ejecución cuando detecta que los datos de entrada son inválidos, antes de que causen un error más difícil de rastrear.

¿Cuándo usarlo?

🛡️
Al inicio de una función, para validar los parámetros que recibe
➗
Cuando una condición haría la operación matemáticamente inválida (ej. divisor = 0)
🏷️
Cuando ninguna excepción de Python describe el error; crea una propia

Jerarquía de excepciones Python

BaseException
Exception
ValueError — valor correcto tipo, contenido inválido
TypeError — tipo de dato incorrecto
ZeroDivisionError — división entre 0
DivisorNegativoError — propia
KeyboardInterrupt
SystemExit
❌ Sin raise

Si pasas dividir(10, 0) sin validación, Python lanza el error en la línea de la división — lejos de donde está el problema real.

✅ Con raise

La función valida primero y lanza ValueError con un mensaje claro. El error señala exactamente qué salió mal y dónde.

logging

registrar · diagnosticar · auditar

¿Qué es?

Módulo de Python que guarda mensajes técnicos en un archivo (o consola) para que el programador pueda diagnosticar errores, sin mostrar esa información al usuario final.

¿Cuándo usarlo?

🔍
Para registrar qué operaciones se ejecutaron y con qué valores
🚨
Para guardar el detalle técnico de un error sin mostrárselo al usuario
📊
En aplicaciones reales donde necesitas auditar el comportamiento del sistema

Niveles de severidad (menor → mayor)

DEBUGDetalles de desarrollo (valores, pasos)
INFOConfirmación: operación completada
WARNINGAlerta: algo inusual pero no crítico
ERRORUna función falló, requiere revisión
CRITICALFallo grave, el sistema puede detenerse
❌ Nunca hagas esto

Mostrar el error directamente al usuario con print(e) expone detalles técnicos que no le son útiles y pueden revelar la estructura interna del programa.

✅ Buena práctica

Usa logging.error() para guardar el detalle técnico y print() solo para un mensaje genérico y amigable al usuario.
