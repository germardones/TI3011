🚨 Errores y Excepciones

Captura y maneja errores de forma segura, evitando exponer información sensible del sistema. Aprende las mejores prácticas orientadas a seguridad.

excepciones_seguras.py 
# INSEGURO: captura genérica expone info del sistema
try:
    archivo = open(ruta, "r")
    datos = int(input("Número: "))
except Exception as e:
    print(f"Error: {e}")  # ¡expone detalles internos!

# ─────────────────────────────────────────────────

# SEGURO: captura específica + mensaje controlado
try:
    with open(ruta, "r") as archivo:
        contenido = archivo.read()
    numero = int(input("Número: "))
except FileNotFoundError:
    print("Archivo no encontrado.")   # mensaje genérico
except ValueError:
    print("Ingresa un número válido.")
except PermissionError:
    print("Sin permiso para acceder.")
finally:
    print("Operación finalizada.")
def validar_edad(edad):
    if not isinstance(edad, int):
        raise TypeError("La edad debe ser un entero.")
    if edad < 0 or edad > 150:
        raise ValueError("Edad fuera de rango válido.")
    return edad

def dividir(a, b):
    if b == 0:
        raise ZeroDivisionError(
            "No se puede dividir por cero.")
    return a / b

# Excepciones propias para validaciones específicas
class ErrorValidacion(Exception):
    pass

def procesar_rut(rut):
    if not rut.strip():
        raise ErrorValidacion(
            "El RUT no puede estar vacío.")
    if len(rut) < 8:
        raise ErrorValidacion(
            "Formato de RUT inválido.")
import logging

# Configuración segura del logger
logging.basicConfig(
    level=logging.WARNING,
    format='%(asctime)s - %(levelname)s - %(message)s',
    filename='app.log'  # guardar en archivo
)

try:
    resultado = 10 / 0
except ZeroDivisionError:
    # Log técnico → al archivo (no al usuario)
    logging.error("División por cero detectada.")
    # ❌ NO: logging.error(f"Error: {e} datos: {datos}")
    print("Ocurrió un error interno.")  # vago al usuario

# Niveles de logging (de menor a mayor severidad)
# DEBUG → INFO → WARNING → ERROR → CRITICAL
logging.debug("Mensaje de depuración")
logging.info("Operación completada")
logging.warning("Alerta: recurso bajo")
logging.error("Error recuperable")
logging.critical("Error crítico del sistema")

try / except

captura específica · never catch all · finally siempre

Capturar excepciones genéricas (Exception) puede ocultar errores reales y exponer información sensible. Siempre captura lo más específico posible.

❌ Anti-patrón peligroso

Usar except Exception as e: print(e) imprime trazas del sistema al usuario, lo que puede revelar rutas de archivos, versiones, estructura interna.

✅ Patrón seguro

Captura FileNotFoundError, ValueError, PermissionError por separado. Muestra mensajes genéricos al usuario; registra los detalles en un log interno.

🎯
Captura específica — identifica qué puede fallar y captura solo eso
🔒
Mensajes genéricos al usuario — no reveles rutas, variables o versiones
🧹
finally — garantiza limpieza de recursos siempre (archivos, conexiones)
📁
with open() — cierra archivos automáticamente aunque haya excepción

Lanzar excepciones de forma controlada permite validar datos al entrada, garantizando que el programa no continúe con datos inválidos o peligrosos.

logging

separar log técnico del mensaje al usuario · niveles

El módulo logging permite registrar eventos del sistema de forma segura, sin exponer información al usuario final.

Niveles de severidad

DEBUGInformación de depuración detallada
INFOConfirmación de operación normal
WARNINGAlerta de comportamiento inesperado
ERRORError grave, función falla
CRITICALError crítico, sistema puede detenerse
❌ Nunca hagas esto

Loggear datos del usuario, contraseñas, tokens o trazas completas con variables sensibles: logging.error(f"datos: {datos_usuario}")

✅ Forma correcta

Loggear solo el tipo de error y contexto mínimo necesario. Al usuario: mensaje genérico. Al log: evento sin datos privados.
