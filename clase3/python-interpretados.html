🔄 Lenguajes Interpretados

Comprende cómo los lenguajes interpretados ejecutan el código de forma diferente a los compilados, y por qué Python es uno de los más populares de esta categoría.

Interpretado

El código fuente se traduce y ejecuta línea por línea en tiempo real mediante un intérprete.

Ventajas y desventajas

Ventajas

  • Desarrollo más rápido y ágil — no es necesario compilar para probar cambios.
  • Portabilidad — el mismo código corre en cualquier sistema donde esté el intérprete instalado.
  • Depuración más sencilla — los errores se muestran en la línea exacta donde ocurren.
  • Interactividad — se puede ejecutar código en tiempo real desde la consola (REPL).
  • Tipado dinámico — no es necesario declarar explícitamente los tipos de variables.

Desventajas

  • Velocidad de ejecución menor — la traducción en tiempo real agrega sobrecarga.
  • Mayor uso de memoria — el intérprete debe estar cargado en RAM durante la ejecución.
  • No apto para tareas de tiempo real crítico (sistemas embebidos de baja latencia).
  • El código fuente está expuesto — no genera un ejecutable binario nativo.
  • Errores de tipo solo se detectan en ejecución, no en compilación.
